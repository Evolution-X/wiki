# Setting Up a Custom OTA Server for Evolution X Unofficial Builds

This guide explains how to set up a custom OTA (Over-The-Air) server for your Evolution X build.

## Prerequisites

- A GitHub account
- A storage server (e.g., Sourceforge, Cloudflare R2, etc.) for hosting your builds
- A properly configured Android build environment

## Fork the Required Repositories

Fork the following repositories from the official Evolution X GitHub account:

- [Packages_apps_Updater](https://github.com/Evolution-X/packages_apps_Updater)
- [OTA](https://github.com/Evolution-X/OTA)

## Modify Updater fork

In the **packages_apps_Updater** repository, update the OTA server URL references in the `app/src/main/res/values/strings.xml` file. Replace the official Evolution X URLs with your forked OTA repository details.

For example, change:

```xml
<string name="updater_server_url" translatable="false">
  https://raw.githubusercontent.com/Evolution-X/OTA/vic/builds/{device}.json
</string>

<string name="menu_changelog_url" translatable="false">
  https://raw.githubusercontent.com/Evolution-X/OTA/vic/changelogs/<xliff:g id="device_name">%1$s</xliff:g>.txt
</string>
```

To (replace (username) to your GitHub username):

```xml
<string name="updater_server_url" translatable="false">
  https://raw.githubusercontent.com/(username)/OTA/vic/builds/{device}.json
</string>

<string name="menu_changelog_url" translatable="false">
  https://raw.githubusercontent.com/(username)/OTA/vic/changelogs/<xliff:g id="device_name">%1$s</xliff:g>.txt
</string>
```

Apply the changes to your project folder, you can cherry-pick them
```bash
git remote add Updater https://github.com/(username)/Packages_apps_Updater.git
git remote update Updater
git cherry-pick (your-commit-hash)
```

## Build and Setup Your OTA Repository

Build and upload your ROM

- Create your Evolution X build using your Android build environment.

```bash
m evolution
```

- Ensure the build is successful and all required files are generated.
- Upload the Evolution X zip file and the needed .img files to install in the target device

Upload Files to Your OTA Repository

1. In your forked **OTA** repository, remove all the existing JSON and TXT files.
2. Locate the JSON file generated by your build at: out/target/product/(device)/(device).json
3. Copy this JSON file into the "builds" folder of your OTA repository.
4. Get yout Evolution X build download link
5. Edit the JSON file as necessary to fill in all the required fields for your OTA update.


Configure Changelogs
1. Inside the "changelogs" folder, create a new file named (device).txt where (device) is your device identifier.
2. Add the following content to the (device).txt file:

```txt
This is Android 15 QPR1 with February 2025 security patches

Notes:
==============================
- LineageOS-based source. Signed build.
- Make sure to be on (device) firmware first.
- Clean flash is required if coming from March 2024 or older builds.
- Always dirty flash at your own risk. If your device bootloops, revert to an older build that boots.
- (Root advice)
- Play Integrity passes, RCS works (As of mm/dd/yyyy).
- Includes (brand) camera.

Device changes:
==============================
* (List all changes for your device, including modifications to the device kernel and vendor files.)

Notable ROM changes:
==============================
* (List all major changes made to the Evolution X project repositories.)
```

Configure Changelogs
1. Commit all the changes made to your OTA repository with a descriptive commit message.
2. Push your changes to your fork on GitHub to update your OTA server with the latest build and changelog information.
